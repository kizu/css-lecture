Вёрстка —
и фасад, и фундамент

Фасад

За внешний вид сайтов отвечает именно вёрстка (HTML и CSS):

  • расположение, размер, цвет элементов;
  • состояния элементов, такие как стиль ссылок при наведении курсора и т.п.;
  • работа с неизвестным количеством содержимого элементов;
  • адаптируемость к разным размерам окна браузера;
  • типографика, и многое, многое другое.

Фундамент

В основе фронтенда лежит вёрстка.

  • Разметка и данные: HTML.
  • Оформление: CSS и изображения.
  • Поведение: CSS и JS.

Существуют «нейтральные» по значению HTML-теги <div> и <span>.

CSS

Всё самое сложно в вёрстке
происходит именно в CSS

CSS — декларативный язык оформления

Синтаксис CSS

a:link {             Правило    color: blue;}a:hover {            ← Селектор    color: red;      ← Декларация    background: lime;}  ↑ Свойство  ↑ Значение

CSS декларативны

  1. Находим то, что нужно оформить.
  2. Оформляем.
  3. Всё.

Если для одного свойства на элементе матчится несколько деклараций, выбирается нужная в соответствии с каскадом и специфичностью.

Каскад

  1. Изначальные браузерные стили.
  2. Пользовательские стили (расширения).
  3. Авторские стили (<style> и прилинкованные .css-файлы).
  4. Инлайновые стили в HTML (<span style="margin: 10px">).
  5. CSS-анимации перебивают любые обычные стили.
  6. !important авторских стилей.
  7. !important инлайновых стилей.
  8. !important пользовательских стилей.

Специфичность

От наименее значимых к наиболее:

  1. Универсальный селектор: *.
  2. Селекторы по элементам: li { … }.
  3. Почти все остальные селекторы: .foo, :hover и т.д.
  4. Селектор по уникальному идентификатору: #Main.

И ещё есть :not(), принимающий специфичность аргумента.

Специфичность

Каждый класс специфичности считается независимо.

  • ul li p a em span всегда меньше, чем просто .foo.
  • .foo.bar:hover .baz:nth-child(2) меньше, чем #Main.
  • Можно увеличивать специфичность класса: .foo.foo.foo больше, чем .foo.bar.
  • Если и каскад и специфичность одинаковы, то применяется последняя по порядку декларация.

Обработка ошибок в CSS

CSS не особо чувствительны к ошибкам*.

  • Ошибка в селекторе: пропускается всё правило.
  • Ошибка в свойстве или значении: пропускается декларация.

* «Ошибка» — с точки зрения браузера, по факту это может быть правильный код, пропуск браузером которого будет ожидаем.
